## 0. 요약 (Executive Summary)

2026-05-18 시점의 P0/P1 작업으로 **테스트 안전망(vitest 11종 + CI verify job)**, **DB 인덱스 5종**, **R2 incremental cache 활성화**, **침묵 catch 3건 수정**, **D1 UNIQUE 헬퍼 채택**이 이미 해소됐다. 그 결과 코드 품질·보안·배포 측면은 양호하다.

그러나 **기능 추가/제거 시 누적 비용을 키우는 구조적 부채**가 잔존한다. 그중 임팩트가 큰 5건은 다음과 같다.

1. **어드민 보일러플레이트 (15개 페이지 동일 패턴)** — 새 콘텐츠 타입 추가 시 8개 이상 파일을 동시 수정해야 함

3. **DB FK 제약 0개 + JSON 블롭 컬럼** — 데이터 정합성 보장 불가, 기능 폐지 시 고아 레코드 잔존
4. **도메인 SSOT 불일치 (`news.jung-youl.com` ↔ `www.jungyoul.net`)** — 도메인 통합 시 다수 파일 일괄 수정 필요

## 3. 기능 추가/제거 시나리오 영향도

각 시나리오는 **(1) 변경 대상 파일** / **(2) 추정 소요** / **(3) 위험 요소**를 명시.

### 시나리오 1 — 새 콘텐츠 타입 추가 (예: 팟캐스트)

**(1) 변경 대상 파일 (최소 8개)**

1. `src/db/schema.ts` — `podcasts` 테이블 정의
2. `drizzle/0010_*.sql` — 마이그레이션 생성
3. `seed.sql` — CREATE TABLE 중복 (선택, 그러나 SSOT 일관성)
4. `src/lib/validation.ts` — `insertPodcastSchema` / `updatePodcastSchema`
5. `src/app/api/admin/podcasts/route.ts` + `[id]/route.ts` — CRUD
6. `src/app/api/podcasts/route.ts` — 공개 GET
7. `src/app/admin/podcasts/page.tsx` — 어드민 UI (300+ LOC 보일러플레이트)
8. `src/components/inline-edit-modal.tsx:30, 37, 44` — `apiMap`/`titleMap`/`fetchMap` 추가
9. `src/components/auth-provider.tsx:5` — `EditModalType` 유니온 확장

11. `src/lib/mappers.ts` — DB→API 변환
12. `src/lib/default-nav.ts` 또는 헤더 메뉴 — 링크 노출

**(2) 추정 소요**: 3~4일 (테스트 작성 시 +1~2일)

**(3) 위험 요소**
- `InlineEditModal` 4곳 누락 시 빌드 통과하지만 런타임 모달 깨짐
- `auth-provider` 유니온 확장은 타입 체크에서만 잡힘 — 테스트 부재로 회귀 발견 지연
- 어드민 페이지 보일러플레이트 ~300 LOC을 매번 손으로 복사 → 오타·필드 누락 가능
- DB 인덱스 누락 시 트래픽 증가 후 풀스캔 (인덱스 부재가 P0로 해소됐으나 신규 테이블엔 재발 가능)

**완화책**: `useAdminResource<T>` 훅 + `<AdminTable>` 컴포넌트 도입 (P1)으로 페이지 LOC 70% 감소 가능

### 시나리오 2 — 어드민 권한 세분화 (편집자/관리자 분리)

**(1) 변경 대상 파일 (30+ 개)**

- `src/app/api/admin/**/route.ts` (30+ 라우트) — 권한 검사 분기

**(2) 추정 소요**: 1~2주

**(3) 위험 요소**
- JWT 토큰 형식 변경 → 발급된 기존 세션 무효화 (강제 재로그인 또는 호환 로직 필요)
